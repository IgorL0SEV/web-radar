## Wildberries.by

**Тип:** SPA (React), рендерит контент через JS

**Подход:** Playwright — единственный надёжный способ

**Почему не API напрямую:**
- `card.wb.ru/cards/v1/detail` → 403
- `catalog.wb.ru/catalog/v1/card` → 403
- `search.wb.ru/exactmatch/ru/common/v18/search` → работает, но только для текстового поиска, не по nmId
- Basket static JSON → 404 (устарело)

**Как собирать данные:**
1. Playwright: загрузить страницу `https://www.wildberries.by/catalog/{nmId}/detail.aspx`
2. Перехватить API-ответ `/__internal/u-card/cards/v4/detail` — из него берём: название, бренд, рейтинг, отзывы, остатки, базовую цену
3. **Цену со скидкой брать из отрендеренной страницы, НЕ из API**
4. Парсить `page_prices[0]` — формат `"29,08 ƃ 61,77 ƃ"` → первое число = цена со скидкой, второе = базовая

**Почему цена со скидкой — только со страницы:**
API `product` поле даёт цену без региональной наценки WB.by (~11% ниже реальной).
Базовая цена `basic` из API совпадает со страницей — её можно брать оттуда.

**Цены:** в копейках BYN из API (делим на 100). На странице — уже в BYN с разделителем запятая.

## Onliner.by

**Тип:** Открытый REST API

**Подход:** Прямые HTTP-запросы, Playwright не нужен

**Эндпоинты:**

| Что | URL |
|---|---|
| Карточка товара | `https://catalog.api.onliner.by/products/{key}` |
| Предложения магазинов | `https://shop.api.onliner.by/products/{key}/positions` |
| Информация о магазине | `https://shop.api.onliner.by/shops/{shop_id}?include=full` |

**Ключевые правила:**
- Поле названия магазина — **`title`**, не `name`
- Ссылка на товар — из поля `html_url` в ответе API продукта
- Ссылка на цены — `{html_url}/prices` — **это основная ссылка для пользователя**, ведёт на страницу со всеми предложениями
- Ссылка на магазин `https://{shop_id}.shop.onliner.by` — **не полезна** (страница магазина без привязки к товару). Не показывать.
- Показывать топ-3 предложения с названиями магазинов, ценами, доставкой, гарантией
- **Главная полезная ссылка для пользователя** — `{html_url}/prices`, страница со всеми предложениями. Прямой ссылки «товар у конкретного продавца» API не даёт. Только `html_url` на карточку товара.

**Позиции (offers):**
- `positions.primary` — массив, сортировать по `position_price.amount`
- Кэшбэк: `cobrand_info.cashback.amount` / `cobrand_info.cashback_percentage`
- Доставка: `delivery.pickup_point.price_text`
- Распродажа: `on_sale: true`

## Общие правила

1. **SPA-сайты → Playwright первым делом.** Не тратить время на curl/requests, если API отдаёт 403.
2. **Открытые API → прямые запросы.** Onliner API открыт — Playwright не нужен.
3. **Конфиг принципов — отдельно от данных.** Товары в `products.yaml`, правила в `SOURCE_CONFIGS.md`.
4. **Цены проверять на достоверность.** Если API отдаёт цену, которая расходится со страницей — доверять странице.
5. **Названия магазинов — `title`, не `name`.** Урок из Onliner API.
